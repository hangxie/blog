Long story short – and stupid me, obvious a while back whenever I applied my patch to nagios.cfg, I didn’t comment out the first\u00a0process_performance_data=0, and obviously second\u00a0process_performance_data=1 didn’t overwrite the first one. Also, even with\u00a0process_performance_data=0 which disable performance data processing, nagios still processes performance data every time it got restarted, and this cause all mysterious problems.<\/p>\n

I do believe nagios behaves wrong in this case – it’s a common sense that configuration can be overwritten by later one with same key, and if configuration tells you don’t do something, DON’T do it even in edge cases, otherwise people will get confused and made debugging really difficult.<\/p>\n","protected":false},"excerpt":{"rendered":"
