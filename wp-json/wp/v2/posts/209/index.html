I just checked in new codes of PPCam to SVN on sourceforge.net<\/a>, I guess I won’t spend too much time on visual effect in the future, so what we have now is most likely what we have all the time:<\/p>\n

    \n
  • no Windows gadget, i.e. no toolbox, no menu, no border, no title bar<\/li>\n
  • semi-transparent effect, means you can still read content on windows below PPCam<\/li>\n
  • Moving by any point, means you can click on any position of the window and drag & drop the window to the new place, this is a must as it does not have title bar for this<\/li>\n<\/ul>\n

    Note that this is data driven \ud83d\ude09 , I learnt the idea from previous music player project – you can just provide a bmp file, and the application will take top-left point as “transparent sample”, then any points in the bmp that has the same color as top-left point will be treated as transparent. This feature makes really easier to change the look and feel.<\/p>\n

    Other than these visible changes, that are some codes related changes, may or may not be that interesting:<\/p>\n

      \n
    • Added win32api.ads to include those Win32 APIs that the project needs to use, but are missing in win32ada binding<\/li>\n
    • PPCam now becomes a dummy top level package, it used to have those data conversion functions but obviously I can use those from win32ada<\/li>\n
    • Added a bmp to the source, this is the bmp I used to debug\/demo<\/li>\n
    • I’m trying to turn some functions to procedures whenever I don’t need a return code, I think this is another C habit needs to be changed<\/li>\n<\/ul>\n

      There will be couple of parts of features to be done in the coming days<\/p>\n

        \n
      1. Include vfw features, I haven’t got any idea so far<\/li>\n
      2. Pop-up menus for the interface to let user enter user name\/password\/server name, etc<\/li>\n
      3. Network, UDP only, then later on I need to have P2P based on it<\/li>\n
      4. Server, another GNAT Ada program that will be running on a server to handle authentication, port discovery, etc<\/li>\n<\/ol>\n

        I believe this project will be running pretty long thinking of how much I have to learn, if I can make a workable version by June I will be super happy.<\/p>\n

        I will try to check in at least every week so to make the project active, I will also post documents (svn first, web site later) so to make it possible for others to understand the whole project well.<\/p>\n","protected":false},"excerpt":{"rendered":"
